1. Atoms
-> 컴포넌트 디자인과 해당 컴포넌트에 기대되는 가장 기본적인 기능만 작성된 최소단위(위치값은 X)
-> 아무데서나 불러도 사용할 수 있도록 유연하게 작성되어야 한다.
-> css 요소는 무조건 styled-component로 작성한다.
-> 자주 변경될 여지가 있는 디자인 요소는 props로 받아서 처리할 수 있도록 작성한다.

2. Molecules
-> 독립적으로 사용되기 힘들지만 그 자체로 독립적인 역할이 있는 컴포넌트 
-> 보통 Oraganism을 구성하는 요소 중 하나이다.
-> 여기서부터는 class 파일을 통해 css 요소를 작성한다.

3. Organisms
-> 사용자에게 의미와 역할이 존재하는 단계의 컴포넌트
-> 컴포넌트의 목적이 뚜렷해지는 단계이다.

4. Templates
-> 개미의 머리, 가슴, 배처럼 페이지를 나누는 큼직한 구분단위
